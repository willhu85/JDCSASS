简介

为什么创建这个框架？

由于所处无线设计部门的工作性质，会有涉及到平台、APP内嵌、H5、iPad、各种PC端和移动端的活动。往往每个项目都需要一个类似这样的目录，然后再把CSS、JS、模板HTML复制进去。然后对于不同类型的项目要区分不同类型的JS&CSS&HTML,作为一个合格的coder是不是已经觉得头大了。那就对了，因为在这个粘贴复制的过程中会出现大量的问题，比如原始版本的CSS不一致，写好的代码不能复用。移动端各种各样的前缀，各种新特性。再加上组员负责的项目不同，无法及时的更新信息。会导致有重复的工作和代码BUG。这时候，一个基本的CSS框架的重要性就体现出来了。大家共同维护核心部分，支持大部分的CSS3和预定义一些特殊的属性，然后组员们在此基础上在写针对每个项目不同的东西，这样就可以把这种兼容适配的时间节省下来，去完成复杂的业务逻辑了。目前JDCSASS只是完成了最基本的CSS部分的Core部分，后续会不断的完善CSS样式组件，默认图标，JS组件等。最终目标是能成为一个适用于PC、移动的类似bootstrap的框架。
从此工作效率提高，升职、加薪、担任总经理，出任CEO、赢取白富美(不能告诉老婆)、走向人生巅峰，想想都有些小激动呢。

Core

Core有以下几个SCSS组成:
config: config 目的是用于定义一些初始的变量，如颜色，基本字体，浏览器配置等等。
css3: css3集合了常用CSS3的mixin，如动画、过渡、变形等
mixin: mixin集合了常用的一些CSS组合属性，例如fix、ellipsis文字溢出，透明、三角形等等。
media-queries: media-queries顾名思义，就是放了一些媒体查询的东东
reset: CSS reset 用了normalize.css，并对其进行了一些针对性的补充 另外还有三个文件，是用于不同平台调用使用
jdc-pc: 针对PC端
jdc-m: 针对移动端
jdc-m-active: 针对移动端的运营活动，用了一个简略版的reset

mixin 范例 [ ]内为选填 说明
transition transition([width 1s ease-in]) 若无参数则默认为transition(all 0.3s ease)
transition-property transition-property(cssName) transition-property为CSS过渡属性名称
transition-duration transition-duration(times) transition-duration为CSS过渡持续时间
transition-timing-function transition-timing-function(motions) transition-duration为CSS过渡效果
transition-delay transition-delay(times) transition-delay为CSS过渡延时
animation animation(animation) eg: animation(test 5s infinite)，配合keyframes使用
animation-name animation-name(name) animation-name为动画的名称
animation-duration animation-duration(duration) animation-duration为动画所花费的时间
animation-timing-function animation-timing-function(timing-function) animation-timing-function为动画的速度曲线(linear,ease,ease-in,ease-out)
animation-delay animation-delay(delay) animation-delay为动画开始前的延迟
animation-iteration-count animation-iteration-count(iteration-count) animation-iteration-count为动画播放的次数
animation-direction animation-direction(direction) animation-direction为是否反向播放动画
(normal默认值,动画应该正常播放。alternate，动画应该轮流反向播放)
animation-fill-mode animation-fill-mode(fill-mode) animation-fill-mode为结束时动画停留的位置
none:不改变默认行为
forwards:当动画完成后，保持最后一个属性值（在最后一个关键帧中定义）。
backwards:在 animation-delay 所指定的一段时间内，在动画显示之前，应用开始属性值（在第一个关键帧中定义）
both:向前和向后填充模式都被应用
backface-visibility backface-visibility([visiblity]) backface-visibility用于3D效果时是否显示背面
hidden:背面是不可见的,默认值
visible:背面是可见的
box-shadow box-shadow(shadow) eg:box-shadow(1px 1px 3px #CCC, inset 1px 1px 3px #CCC)
box-sizing box-sizing(box-model) 规定两个并排的带边框的框,content-box|border-box|inherit
placeholder placeholder(color) placehoder text 颜色
transformations类

scale3D scale3D(ratio...) ...为多参数，下同。eg：scale3D(1,2,2)
scaleX|scaleY|scaleZ scaleX(ratio)|scaleY(ratio)|scaleZ(ratio)
skew skew(degrees...)
skewX|skewY skewX(degrees)|skewY(degrees)
translate translate(px...)
translate3D translate3D(px...)
translateX|translateY|translateZ translateX(px)|translateY(px)|translateZ(px)

rotate3D rotate3D(degrees...)
rotateX|rotateY|rotateZ rotateX(degrees)|rotateY(degrees)|rotateZ(degrees)
